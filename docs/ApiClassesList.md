# External API

- Histogramator
- SourceFlowGenerator

# Internal API

- ConfigurationParser
- ChunkAggregator
- ChunkStorage
- ChunkQueue
- ChunkParameterBuilder
- HistogramBuilder

## Histogramator

- Фасад ля построения гистограммы на основе потока данных
- Использует `HistogramBuilder`, `ConfigurationParser`

## SourceFlowGenerator

- API для генерации потока исходных данных
- Реализация по умолчанию:
- Генерирует поток исходных данных для построения гистограммы

> [!INFO]
> Вынесен во внешнее API, для возможности использования различных типов данных при построении гистограммы\
Реализация потока должна быть осуществлена пользователем через имплементацию интерфейса

## ConfigurationParser

- API для работы с конфигурацией
- Реализация по умолчанию преобразует формат конфигурационного файла в настройки параметров гистограммы и чанков

## ChunkAggregator

- API для агрегации элементов, поступающих из SourceFlowGenerator в чанки (гистограммы с константными границами)
- Точки взаимодействия:
- `Store`
- `Queue`
- `SourceFlow`
- `HistogramBuilder`
- В реализации по умолчанию по мере поступления данных выполняется следующий алгоритм:
- Определяем chunk для записи "точки" (минимальной единицы данных)
- Добавляем "точку" к чанку (минимальной гистограмме статического размера)
- Подгружаем параметры чанка (гистограммы)
- Добавляем "точку" к нужному бину (определяется математически)

## ChunkStorage

- API для управления хранилищем параметров и состояния чанков
- API для сохранения/получения данных о чанках

## ChunkQueue

- API для работы с очередью сообщений о необходимости аккумуляции чанка в общей гистограмме
- Предполагаются различные реализации в виде персистентных хранилищ, очередей в памяти, key-value хранилищ и пр.

## ChunkParameterBuilder

- API для инициализации параметров общей гистограммы и параметров чанков

## HistogramBuilder

- API для построения общей гистограммы
